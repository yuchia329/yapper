Yapper — 电影解說 pipeline

Architecture map & infrastructure data flow · package yapper

Turns a full-length movie into a ~10–12 min Mandarin funny commentary (Douyin/Kuaishou 电影解說 style): condensed footage from the film plays under an AI-generated voiceover that tells the story. It runs as one CLI — uv run yap run movie.mkv — through 13 deterministic, resumable stages, each reading and writing one validated artifact.

2 · The 13-stage pipeline — artifacts in / out

Each stage is a pure function consuming one typed Pydantic artifact and producing another, cached under artifacts/<movie-slug>/. A stage is skipped if its artifact already exists (unless --force). --until <stage> stops early. Timecodes are always seconds (never frame numbers) so variable-frame-rate films don't drift.

★ The load-bearing idea — clip_ref grounding

The LLM never invents timestamps. Every scene gets a stable clip_id (e.g. clip_0042) in the SQLite index built at s04. The model emits clip_refs[] per narration line; db.resolve_clip_refs() is the single choke point that turns ids into real timecodes — and raises on any id it doesn't recognize, so a hallucinated reference is caught at s08 before a cent is spent on TTS. Editing is audio-driven: each segment's screen time equals the measured voiceover duration, so footage is conformed to the narration, never time-stretched to fit.

5 · Things worth knowing (verified against source)

⚠ Stale documentation — the "Claude" references are wrong

  • The README says "Claude Opus 4.8 (API)" and a pipeline.py docstring says ANTHROPIC_API_KEY, but the code uses the OpenAI SDK against an OpenAI-compatible endpoint — default MiniMax-M3 at api.minimax.io/v1. There is no anthropic dependency in pyproject.toml.
  • The real env var read by the back half is LLM_API_KEY (via require_env). The names MINIMAX_API_KEY (in cli.py error text & a toml comment) and ANTHROPIC_API_KEY (docstring) are not read anywhere.
  • Structured output is NOT a native "Anthropic structured-output API" — it's response_format={"type":"json_object"} with the JSON schema inlined into the prompt text; the client then strips <think> blocks/fences and parses.

ℹ Minor wiring details

  • --until tokens are the short names: ingest, audio, asr, shots, scenes, understand, script, budget, tts, edl, subs, render.
  • s09 passes voice_id = [tts].provider → resolves to "cosyvoice2" (provider name reused as voice id). speed is never sent (client default 1.0).
  • If ASR_SERVER_URL is unset and a film has no embedded subs, the pipeline continues without dialogue (empty transcript) rather than failing.
  • Cached re-renders need no API key or TTS tunnel — the LLM client is built only when a script stage must actually run.
